# JTL Comparator

Локальное веб-приложение для сравнения результатов нагрузочного тестирования JMeter.

Актуальный граф кода и связей модулей: [CODE_GRAPH.md](./CODE_GRAPH.md)

## Возможности

- Загрузка двух `.jtl` файлов с произвольными именами прогонов
- Автоматическое определение Transaction Controller vs HTTP-сэмплеров
- Агрегация метрик: Samples, Avg, p95, p99, RPS, Error Rate
- Сравнительная таблица с цветовой подсветкой деградации/улучшения
- Настраиваемые правила подсветки дельт (time/RPS/error rate) прямо в UI
- Drag & Drop загрузка файлов
- Sticky header при скролле
- Итоговая строка Summary
- Экспорт: Markdown (для Confluence) и CSV

## Правила подсветки дельты

Пороги можно менять в блоке **«Правила подсветки дельты»** на странице перед запуском сравнения.
Ниже указаны дефолтные значения.

### Что такое дельта

Для каждой метрики считается процент изменения между Run 1 и Run 2:

`Δ% = (Run2 - Run1) / Run1 * 100`

- `Δ Avg%` — изменение среднего времени ответа.
- `Δ p95%` — изменение 95-го перцентиля (95% запросов быстрее этого времени).
- `Δ p99%` — изменение 99-го перцентиля (хвост медленных запросов).
- `Δ RPS%` — изменение пропускной способности (запросов/сек).

Если базовое значение `Run1 = 0`, дельта не рассчитывается и показывается `—`.

### Как интерпретировать знак

- Для времени (`Avg/p95/p99`):  
  `+` = хуже (стало медленнее), `-` = лучше (стало быстрее).
- Для `RPS`:  
  `+` = лучше (выше throughput), `-` = хуже.
- Для `Error Rate`:  
  сравнивается разница в процентных пунктах (`Err2 - Err1`), где рост ошибки — ухудшение.

### Настраиваемые пороги в UI

В блоке **«Правила подсветки дельты»** можно менять:

- Время: `time_warning_pct`, `time_critical_pct`, `time_improved_pct`.
- RPS: `rps_warning_drop_pct`, `rps_critical_drop_pct`, `rps_improved_gain_pct`.
- Error Rate: `err_warning_increase_pct`, `err_critical_increase_pct`, `err_improved_decrease_pct`.

Валидация:

- `critical` порог должен быть `>= warning` для времени, RPS и Error Rate.
- Отрицательные значения порогов запрещены.

### Метрики времени (Avg, p95, p99) — деградация = рост

| Δ% | Цвет |
|---|---|
| > +20% | 🔴 Красный (critical) |
| +10% … +20% | 🟡 Жёлтый (warning) |
| −10% … +10% | Нейтральный |
| < −10% | 🟢 Зелёный (improved) |

### RPS — деградация = снижение

| Δ% | Цвет |
|---|---|
| < −20% | 🔴 Красный |
| −20% … −10% | 🟡 Жёлтый |
| > +10% | 🟢 Зелёный |

### Error Rate

| Условие | Цвет |
|---|---|
| Err R2 > Err R1 + 3% | 🔴 Красный |
| Err R2 > Err R1 + 1% | 🟡 Жёлтый |
| Err R2 < Err R1 | 🟢 Зелёный |

### Быстрые примеры

- Время: было `100ms`, стало `120ms` -> `Δ = +20%` -> деградация.
- Время: было `100ms`, стало `85ms` -> `Δ = -15%` -> улучшение.
- RPS: было `50`, стало `40` -> `Δ = -20%` -> деградация по пропускной способности.
- Error Rate: было `1.0%`, стало `2.5%` -> `+1.5 п.п.` -> warning (по умолчанию).

## Как пользоваться в интерфейсе

1. Загрузите `Run 1` и `Run 2`.
2. (Опционально) измените пороги в блоке **«Правила подсветки дельты»**.
3. Нажмите **«Сравнить»**.
4. Наведите на ячейки `Δ%`, чтобы увидеть формулу и пояснение.
5. Выгрузите отчет через **Markdown** или **CSV**.

## Summary

- Строка `SUMMARY (avg)` считается как взвешенное среднее по транзакциям.
- Вес каждой транзакции равен `Samples`, чтобы итог не искажался из-за редких label.
