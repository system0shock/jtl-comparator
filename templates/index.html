Как пользоваться инструментом
  1. Задайте имена прогонов и загрузите два JTL-файла (Run 1 — база, Run 2 — новая версия).
  2. При необходимости откройте блок правил и настройте пороги подсветки под ваш SLA.
  3. Нажмите «Сравнить» и проанализируйте таблицу: время (Avg/p95/p99), RPS и Error Rate.
  4. Наведите курсор на ячейки Δ%, чтобы увидеть формулу расчета и интерпретацию «лучше/хуже».
  5. Экспортируйте результат в Markdown или CSV для отчета.

Для метрик времени рост значения — это ухудшение, а для RPS рост — улучшение.

Как работает анализ

Экспериментальная функция. Результат носит справочный характер и не заменяет профессиональный разбор инженера по нагрузочному тестированию. Модель может ошибаться, особенно при малом числе сэмплов или нетипичных распределениях.

В модель передаётся только агрегированная сводка — исходные JTL-файлы не покидают компьютер. Данные структурируются по разделам:

  • Сводные метрики — Avg, p95, p99, RPS, Err% по всем транзакциям.
  • Распределение — сколько транзакций попало в каждый класс severity.
  • Критические / Предупреждения / Улучшения — топ транзакций каждого класса, отсортированных по |Δ Avg%|.
  • Новые / выбывшие — транзакции, присутствующие только в одном прогоне.

Число строк в каждом разделе зависит от выбранной модели:
